Yes, there are several ways to tell `cc` (or GCC/clang) not to search for default libraries:

## 1. **`-nostdlib`** (Most common)
Completely disables both the default library search paths and the standard startup files:
```bash
cc -nostdlib -o program program.c
```

## 2. **`-nodefaultlibs`**
Disables only the default libraries (not startup files):
```bash
cc -nodefaultlibs -o program program.c
```
The linker won't automatically link against `libc`, `libgcc`, etc.

## 3. **`-nostdlib` vs `-nodefaultlibs`** difference:
- `-nostdlib`: No startup files + no default libraries
- `-nodefaultlibs`: Still links startup files (crtbegin.o, etc.) but no default libraries

## 4. **`-nolibc`** (GCC-specific)
Disables linking against the C library specifically:
```bash
cc -nolibc -o program program.c
```

## Example usage:
```bash
# Link only with manually specified libraries
cc -nostdlib -o myprog myprog.o -lmylib

# For freestanding/kernel development
cc -nostdlib -ffreestanding -o kernel kernel.o
```

## Note:
Without default libraries, you'll need to:
- Provide your own implementation of C runtime functions
- Implement system call wrappers
- Define `_start` instead of `main`
- Link necessary libraries manually with `-l` flags

The most commonly used is `-nostdlib` when you want complete control over the linking process.
